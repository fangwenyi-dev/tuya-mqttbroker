> ESP32-S3 智能开窗器 LoRa 网关桥接器，基于 TuyaOpen SDK 接入涂鸦云生态。

## 一、项目概述

本项目运行在 ESP32-S3-WROOM-1-N16R8 上，实现：

1. **内置 MQTT Broker**（mosquitto v2.0.20，端口 1883）—— 接收 LoRa 网关上报的设备状态
2. **协议桥接层**（`app_protocol_bridge`）—— 将 $SH MQTT 协议与涂鸦 DP 双向转换
3. **涂鸦 DP 桥接层**（`app_tuya_bridge`）—— 管理设备-DP 映射，支持位置/控制/电池/风锁模式
4. **TuyaOpen SDK** —— 管理 WiFi 配网、涂鸦云连接、DP 收发、OTA 升级

### 架构数据流

```
[LoRa 子设备（开窗器）]
        ↓ LoRa 无线
[LoRa 网关]
        ↓ MQTT over WiFi ($SH 协议)
        ↓ broker 地址: <ESP32_IP>:1883
[ESP32-S3（内置 mosquitto Broker + 协议桥接 + 涂鸦 DP 桥接）]
        ↓ TuyaOpen SDK（DP 上报 / DP 接收）
[涂鸦云]
        ↓
[涂鸦 App]
```

## 三、涂鸦平台配置（需要你操作）

### 3.1 创建产品

1. 登录 [涂鸦 IoT 平台](https://iot.tuya.com/)
2. 创建产品 → 选择"网关"或"自定义类别" → 通信协议选 WiFi+BLE
3. 获取 **Product ID (PID)** —— 记录下来，后续填入代码

### 3.2 申请授权码

1. 访问 https://platform.tuya.com/purchase/index?type=6
2. 申请 open-sdk 授权码
3. 获取 **UUID** 和 **AuthKey** —— 记录下来，后续填入代码

### 3.3 配置 DP（数据点）

在涂鸦平台为产品创建以下 DP（最多 12 个设备 × 4 DP = 48 个 DP）：

| DP ID | 功能 | 类型 | 传输 | 值范围 |
|-------|------|------|------|--------|
| 1, 5, 9, ..., 45 | 位置状态 | Value | 只读 | 0-100 (0=关闭, 100=打开) |
| 2, 6, 10, ..., 46 | 控制命令 | Enum | 可读写 | 0=停止, 1=打开, 2=关闭, 3=内倒 |
| 3, 7, 11, ..., 47 | 电池百分比 | Value | 只读 | 0-100 |
| 4, 8, 12, ..., 48 | 风锁模式 | Enum | 可读写 | 0=内倒模式, 1=平开模式 |

> 设备 N (0-indexed) 的 DP ID = N×4 + offset + 1

### 3.4 配置 App 面板

在涂鸦平台为产品配置 App 面板，建议：
- 每组 4 个 DP 对应一个开窗器控件
- 位置状态用"进度条"或"百分比滑块"
- 控制命令用"按钮组"（停止/打开/关闭/内倒）
- 电池百分比用"电量图标"
- 风锁模式用"开关"或"枚举选择"

## 八、配网与使用

### 8.1 配网流程

1. 烧录固件后，设备启动，LED 蓝灯慢闪（等待配网）
2. 打开涂鸦 App → 添加设备 → 扫描到设备
3. 输入 WiFi 密码，开始配网（BLE+AP 方式）
4. 配网成功后，LED 蓝灯常亮（已连接涂鸦云）

### 8.2 LoRa 网关配置

1. 获取 ESP32 的 IP 地址（从路由器管理页面或串口日志中查看）
2. 在 LoRa 网关中配置 MQTT broker 地址为 `ESP32_IP:1883`
3. LoRa 网关连接后会自动发送 001 绑定消息
4. ESP32 自动回复绑定确认，并请求设备列表（002）
5. LoRa 子设备自动出现在涂鸦 App 中

### 8.3 添加 LoRa 子设备

1. 在 ESP32 上**双击按键**（2击）—— 启动 LoRa 配对模式（绿灯快闪 3 秒）
2. LoRa 网关进入配对模式，60 秒内可配对子设备
3. 子设备配对成功后，自动出现在涂鸦 App 中
4. 重复此步骤添加更多子设备（最多 12 个）

### 8.4 删除 LoRa 子设备

- **3击按键**：删除所有已绑定的 LoRa 子设备（红灯快闪）
- 设备间会逐个发送解绑命令，间隔 200ms
- 如果网关未回复解绑响应，3 秒后强制本地移除（兜底机制）

### 8.5 重置配网

- **长按 5 秒**：清除 WiFi + 云绑定，重新进入配网模式（红灯快闪）
- **连续 3 次上电**：5 秒内连续上电 3 次，自动清除配网信息

## 十、LED 指示灯说明

单颗 LED（GPIO48），按优先级显示不同颜色指示系统状态。

### 颜色优先级

单闪 > 快闪 > 慢闪 > 常亮 > 关闭；同模式时：红 > 绿 > 蓝。

### 蓝色（WiFi/涂鸦云状态）

| 状态 | 含义 |
|------|------|
| 慢闪（1Hz） | 等待配网 / WiFi 未连接 |
| 快闪（5Hz） | 配网中（TUYA_EVENT_BIND_START） |
| 常亮 | 已连接涂鸦云（TUYA_EVENT_DIRECT_MQTT_CONNECTED） |

### 绿色（LoRa 通信状态）

| 状态 | 含义 |
|------|------|
| 快闪（5Hz） | LoRa 配对模式中（2击触发，3 秒后熄灭） |
| 单闪（50ms） | MQTT 消息收发（LoRa 网关 ↔ ESP32 双向通信） |

### 红色（操作状态）

| 状态 | 含义 |
|------|------|
| 快闪（5Hz） | 删除操作进行中（3击触发）或重置配网中（长按触发） |

## 十一、$SH 协议处理

### 11.1 支持的消息类型

| ctype | 方向 | 功能 | 处理逻辑 |
|-------|------|------|---------|
| 001 | 网关→桥接器 | 网关绑定 | 回复 errcode=0 + uuid，然后请求 002 设备列表 |
| 002 | 网关→桥接器 | 设备列表 | 解析 devices 数组，为每个设备注册涂鸦 DP |
| 003 | 双向 | 配对/解绑 | bind=1 配对成功 → 注册 DP；bind=0 解绑成功 → 移除 DP |
| 004 | 桥接器→网关 | 设备控制 | App 下发 DP → 转换为 $SH 004 w_travel 命令 |
| 005 | 网关→桥接器 | 状态上报 | 解析位置/电池/状态/风锁模式，上报涂鸦 DP |

### 11.2 控制命令映射

| 涂鸦 DP control | 动作 | $SH 004 value | 说明 |
|----------------|------|---------------|------|
| 0 | 停止 | 101 | COMMAND_VALUE_STOP |
| 1 | 打开 | 100 | COMMAND_VALUE_OPEN |
| 2 | 关闭 | 0 | COMMAND_VALUE_CLOSE |
| 3 | 内倒 | 200 | COMMAND_VALUE_TOGGLE（仅 SN 前缀 "5005" 设备） |

### 11.3 风锁模式映射

| 涂鸦 DP wind_lock | 模式 | $SH 004 value | 说明 |
|-------------------|------|---------------|------|
| 0 | 内倒模式 | 0 | rwp_wind_lock_mode=0 |
| 1 | 平开模式 | 1 | rwp_wind_lock_mode=1 |

> 风锁模式仅对 SN 前四位为 "5005" 的设备有效。

### 11.4 电池电压转换

LoRa 网关上报的 battery/voltage 字段为原始电压值（单位：×0.1V），需转换为百分比：

```
百分比 = (原始值 - 80) × 100 / (140 - 80)
```

- 有效范围：80-140（对应 8.0V-14.0V，12V 锂电池）
- 低于 80 → 0%，高于 140 → 100%

## 十二、DP 分配方案

每个开窗器占用 4 个 DP，最多支持 12 个设备（48 个 DP）。

| DP ID | 功能 | 类型 | 值 |
|-------|------|------|-----|
| N*4+1 | 位置状态 | Value | 0-100 (0=关闭, 100=打开) |
| N*4+2 | 控制命令 | Enum | 0=停止, 1=打开, 2=关闭, **3=内倒** |
| N*4+3 | 电池百分比 | Value | 0-100 |
| N*4+4 | 风锁模式 | Enum | 0=内倒模式, 1=平开模式 |

**内倒按钮**：DP control=3 → $SH w_travel value=200（仅 SN 前缀 "5005" 的设备支持）
**风锁模式**：DP wind_lock → $SH rwp_wind_lock_mode（仅 SN 前缀 "5005" 的设备支持）

## 十四、API 适配详情

本项目完全基于 TuyaOpen SDK API 实现，无 ESP-IDF 直接依赖（mosquitto 组件除外）。

### 14.1 核心 API 映射

| 功能 | ESP-IDF API（旧） | TuyaOpen API（新） |
|------|-------------------|-------------------|
| MQTT 客户端 | `esp_mqtt_client_init()` | `mqtt_client_new()` + `mqtt_client_init()` |
| MQTT 发布 | `esp_mqtt_client_publish()` | `mqtt_client_publish()` |
| MQTT 连接 | `esp_mqtt_client_start()` | `mqtt_client_connect()` |
| MQTT 事件循环 | `esp_event_loop_create()` | `mqtt_client_yield()`（手动轮询） |
| 线程创建 | `xTaskCreate()` | `tal_thread_create_and_start()` |
| 互斥锁 | `SemaphoreHandle_t` + `xSemaphoreCreateMutex()` | `MUTEX_HANDLE` + `tal_mutex_create_init()` |
| 消息队列 | `QueueHandle_t` + `xQueueCreate()` | `QUEUE_HANDLE` + `tal_queue_create_init()` |
| 内存分配 | `malloc()` / `free()` | `tal_malloc()` / `tal_free()` |
| JSON | `cJSON_free()` | `tal_free()` |
| 日志 | `ESP_LOGI()` | `PR_INFO()` / `PR_DEBUG()` / `PR_ERR()` |
| WiFi MAC | `esp_wifi_get_mac()` | `tkl_wifi_get_mac(WF_STATION, &mac)` |
| GPIO | `gpio_config()` / `gpio_set_level()` | `tkl_gpio_init()` / `tkl_gpio_write()` |
| 系统重启 | `esp_restart()` | `tal_system_reset()` |
| 时间戳 | `time(NULL)` | `tal_time_get_posix()` |
| 重置配网 | `tuya_iot_reset(&client, ...)` | `tuya_iot_reset(tuya_iot_client_get())` |

### 14.2 关键 API 签名说明

#### 互斥锁（TAL Mutex）

```c
// 类型：MUTEX_HANDLE（不是 THREAD_MUTEX）
MUTEX_HANDLE my_mutex = NULL;

// 创建：tal_mutex_create_init（不是 tal_mutex_create）
tal_mutex_create_init(&my_mutex);  // 传入指针

// 使用
tal_mutex_lock(my_mutex);
// ... 临界区 ...
tal_mutex_unlock(my_mutex);
```

#### 消息队列（TAL Queue）

```c
QUEUE_HANDLE my_queue = NULL;

// 创建：参数顺序是 (句柄指针, 消息大小, 消息数量)
tal_queue_create_init(&my_queue, sizeof(my_msg_t), 10);

// 发送和接收
tal_queue_post(my_queue, &msg, 0);        // 返回 OPERATE_RET
tal_queue_fetch(my_queue, &msg, 1000);     // 返回 OPERATE_RET，OPRT_OK=成功
```

#### MQTT 客户端（TuyaOpen libmqtt）

```c
// 创建和初始化（两步）
void *client = mqtt_client_new();
mqtt_client_config_t cfg = {
    .host = "127.0.0.1",
    .port = 1883,
    .clientid = "my_client",
    .on_connected = my_connect_cb,
    .on_disconnected = my_disconnect_cb,
};
mqtt_client_init(client, &cfg);

// 连接（非阻塞，需单独 task 调用 yield）
mqtt_client_connect(client);

// 发布（返回 msgid，0 表示失败）
uint16_t msgid = mqtt_client_publish(client, "topic", payload, len, qos);

// 事件轮询（必须在独立 task 中循环调用）
mqtt_client_yield(client);

// 清理
mqtt_client_disconnect(client);
mqtt_client_deinit(client);
mqtt_client_free(client);
```

#### 重置涂鸦配网

```c
// TuyaOpen 标准方式（使用 tuya_iot_client_get() 获取全局 client）
// tuya_iot_init() 内部会注册全局 client，tuya_iot_client_get() 返回该实例
tuya_iot_reset(tuya_iot_client_get());
// 调用后触发 TUYA_EVENT_RESET → TUYA_EVENT_RESET_COMPLETE → tal_system_reset()
```

## 十六、初始化流程

`tuya_main.c` 中的 `user_main()` 初始化顺序：

```
 1. cJSON 内存钩子（tal_malloc/tal_free）
 2. 日志初始化（tal_log_init）
 3. KV 存储（tal_kv_init）
 4. 软件定时器（tal_sw_timer_init）
 5. 工作队列（tal_workq_init）
 6. CLI 和授权（tal_cli_init + tuya_authorize_init）
 7. 重置配网计数器（reset_netconfig_start）
 8. 读取授权信息（tuya_authorize_read → 失败则用 tuya_config.h 默认值）
 9. TuyaOpen IoT 客户端初始化（tuya_iot_init）—— 注册全局 client
10. LwIP 初始化（TUYA_LwIP_Init）
11. 网络管理初始化（netmgr_init + NETCONN_WIFI）
12. 配置 BLE+AP 配网（netmgr_conn_set NETCFG_TUYA_BLE | NETCFG_TUYA_WIFI_AP）
13. LED 初始化（蓝灯慢闪：等待配网）
14. DP 桥接层初始化（app_tuya_bridge_init）
15. 获取 MAC 地址作为 bridge_sn
16. 协议桥接层初始化（app_protocol_bridge_init）
17. 按键模块初始化（app_button_init）
18. 启动协议桥接（app_protocol_bridge_start → mosquitto broker + bridge task）
19. 启动系统监控 task
20. 启动 TuyaOpen IoT 主循环（tuya_iot_start）
21. 检查重置配网（reset_netconfig_check → 连续 3 次上电则重置）
22. 进入主循环（tuya_iot_yield）
```

> **关键顺序**：`tuya_iot_start()` 必须在 `reset_netconfig_check()` 之前调用，
> 因为重置检查需要 IoT 客户端已就绪。

## 十八、故障排除

### 编译错误

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `tal_api.h not found` | TuyaOpen SDK 路径未配置 | 确认应用在 `TuyaOpen/apps/tuya_cloud/` 下，执行 `tos.py config` |
| `mosq_broker.h not found` | mosquitto 组件未集成 | 将 `espressif__mosquitto` 复制到 `components/` 目录 |
| `tuya_config_secrets.h not found` | 授权文件未创建 | `cp src/tuya_config_secrets.h.example src/tuya_config_secrets.h` |
| `MUTEX_HANDLE undeclared` | 头文件未包含 | 确认包含 `tal_api.h`（内含 `tal_mutex.h`） |
| `THREAD_MUTEX undeclared` | 使用了旧类型名 | 已修复为 `MUTEX_HANDLE`，请确认使用最新代码 |
| `tal_mutex_create` 未定义 | 使用了旧函数名 | 已修复为 `tal_mutex_create_init`，请确认使用最新代码 |

### 运行时问题

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 设备无法配网 | 授权码未填或错误 | 检查 `tuya_config_secrets.h` 中的 UUID/AuthKey |
| 蓝灯一直慢闪 | WiFi 未连接 | 检查路由器、确认配网操作正确 |
| LoRa 设备不显示 | 网关未连接 broker | 检查 LoRa 网关 MQTT 配置（指向 ESP32 IP:1883） |
| 控制无响应 | 设备未绑定到网关 | 2击触发配对模式，等待设备加入 |
| 内倒按钮不可用 | 设备 SN 前缀非 "5005" | 内倒/风锁仅支持 SN 前缀 "5005" 的设备 |
| 重置后仍连旧 WiFi | 重置未完全执行 | 长按 5 秒或连续 3 次上电，观察红灯快闪 |
| mosquitto broker 崩溃 | 栈空间不足 | 增大 `broker_param.stackDepth`（当前 16KB） |
